# Grade Clicker

## 1. Название проекта
**Grade Clicker** (Кликер оценок) — интерактивное Android-приложение, реализованное на Jetpack Compose. Пользователь нажимает на центральное изображение, получая по `+5` баллов за каждый клик. При достижении определённых порогов (`50`, `70`, `90`, `100`) динамически меняются отображаемая оценка, цвет интерфейса и фоновое изображение. Приложение наглядно демонстрирует работу жизненного цикла Activity и механизмы сохранения состояния при изменении конфигурации устройства.

## 2. Цель работы
Изучить жизненный цикл Activity (Activity Lifecycle) в Android, понять последовательность вызова callback-методов при различных сценариях использования приложения (запуск, сворачивание, возврат, закрытие), а также освоить практические способы сохранения состояния UI при изменении конфигурации (например, повороте экрана) с использованием `rememberSaveable`.

## 3. Ответы на контрольные вопросы

**◦ Какие три метода вызываются при запуске приложения?**  
При первом запуске последовательно вызываются: `onCreate()` → `onStart()` → `onResume()`.

**◦ Чем отличается `onPause()` от `onStop()`?**  
`onPause()` вызывается, когда Activity теряет фокус, но остаётся частично видимой (например, поверх появляется диалог или уведомление). `onStop()` вызывается, когда Activity становится полностью невидимой (пользователь нажал кнопку Home, свернул приложение или перешёл в другое приложение).

**◦ Когда вызывается `onRestart()`?**  
Вызывается только при перезапуске Activity после перехода её в состояние `Stopped`. Например, когда пользователь возвращает приложение на передний план из списка недавних задач после нажатия кнопки Home.

**◦ Что происходит при повороте экрана?**  
Поворот экрана является изменением конфигурации (`configuration change`). По умолчанию Android полностью уничтожает текущую Activity (вызывается `onDestroy()`), а затем создаёт её заново (вызывается `onCreate()`). Без специальных механизмов сохранения это приводит к потере данных (баллов, кликов).

**◦ В чём разница между `remember` и `rememberSaveable`?**  
`remember` сохраняет состояние только в пределах текущей рекомпозиции Composable-функции и теряет данные при пересоздании Activity. `rememberSaveable` сохраняет состояние как при рекомпозиции, так и при изменении конфигурации устройства, автоматически восстанавливая данные после пересоздания Activity.

**◦ Почему `onCreate()` должен вызывать `super.onCreate()`?**  
Этот вызов необходим для передачи управления родительскому классу `AppCompatActivity`, который выполняет базовую системную инициализацию и завершает создание Activity согласно внутреннему жизненному циклу Android. Без этого вызова приложение может работать некорректно или аварийно завершаться.

**◦ Для чего используется класс `Log`?**  
Класс `Log` используется для вывода отладочных сообщений в системную консоль Logcat (`Log.d()`, `Log.i()`, `Log.w()`, `Log.e()`). Он позволяет разработчикам отслеживать последовательность вызова методов жизненного цикла, анализировать поведение приложения в реальном времени и находить ошибки.

**◦ Какой метод вызывается только один раз за время жизни Activity?**  
`onCreate()` — вызывается ровно один раз при первичном создании экземпляра Activity.
